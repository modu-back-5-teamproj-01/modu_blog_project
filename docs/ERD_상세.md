# ERD 상세

![ERD](../static/images/blog_project_diagram.png)
## 1️⃣ users — 사용자 계정 정보
| 필드                      | 의미                  | 사용 목적                        |
| ----------------------- | ------------------- | ---------------------------- |
| id                      | PK, 사용자 고유 ID       | 다른 테이블의 FK 참조, 사용자 식별        |
| username                | 로그인 및 화면 표시용 이름, 고유 | 회원가입/로그인 시 사용, 게시글/댓글 작성자 표시 |
| password_hash           | 비밀번호 해시             | 로그인 인증, 보안상 원문 비밀번호는 저장 X    |
| email                   | 이메일, 고유, 선택적        | 계정 복구, 알림 용도                 |
| bio                     | 자기소개                | 프로필 페이지 표시                   |
| created_at / updated_at | 계정 생성/마지막 수정 시각     | 가입일 표시, 회원정보 변경 내역 관리        |

- **사용 시나리오**: 
    - 회원가입, 로그인/로그아웃, 프로필 조회, 게시글/댓글 작성 시 작성자 정보 확인

## 2️⃣ sessions — JWT 세션 관리
| 필드                     | 의미                     | 사용 목적                    |
| ---------------------- | ---------------------- | ------------------------ |
| id                     | PK, 세션 고유 ID           | 세션 단위 관리                 |
| user_id                | FK, `users.id`, unique | 한 유저당 단일 세션 정책 적용        |
| refresh_token_hash     | Refresh Token 해시       | Access Token 만료 시 재발급 검증 |
| issued_at / expires_at | 세션 발급/만료 시각            | JWT 유효성 확인               |
| revoked                | 로그아웃 여부                | true면 토큰 사용 불가, 단일 세션 구현 |


- **사용 시나리오**:

    - 로그인 시 세션 생성, 토큰 검증, 로그아웃 시 revoked 처리

## 3️⃣ posts — 블로그 게시글
| 필드                                     | 의미             | 사용 목적                  |
| -------------------------------------- | -------------- | ---------------------- |
| id                                     | PK             | 게시글 고유 ID              |
| author_id                              | FK, `users.id` | 작성자 정보 연결              |
| title / content / summary              | 제목, 본문, 요약     | 게시글 내용, AI 요약 기능 활용 가능 |
| view_count                             | 조회수            | 게시글 조회 시 자동 증가         |
| created_at / updated_at / published_at | 작성/수정/게시 시각    | 히스토리 관리, 정렬, 화면 표시     |

- **사용 시나리오**:
    - 게시글 CRUD, 조회수 증가, AI 기능 적용

## 4️⃣ tags — 게시글 태그
| 필드   | 의미        | 사용 목적                    |
| ---- | --------- | ------------------------ |
| id   | PK        | 태그 고유 ID                 |
| name | 태그 이름, 고유 | 게시글 분류, 검색 및 AI 자동 추천 기능 |

- **사용 시나리오**: 
    - 게시글 작성 시 태그 연결, 특정 태그 게시글 조회

## 5️⃣ post_tags — 게시글 ↔ 태그 연결 (N:M)
| 필드                         | 의미             | 사용 목적               |
| -------------------------- | -------------- | ------------------- |
| post_id                    | FK, `posts.id` | 게시글 참조              |
| tag_id                     | FK, `tags.id`  | 태그 참조               |
| (post_id, tag_id) [unique] | 복합 유니크 인덱스     | 같은 게시글에 같은 태그 중복 방지 |


- **사용 시나리오**:
    - 게시글 여러 태그 연결, 태그별 게시글 조회

## 6️⃣ comments — 댓글
| 필드                      | 의미                | 사용 목적                    |
| ----------------------- | ----------------- | ------------------------ |
| id                      | PK                | 댓글 고유 ID                 |
| post_id                 | FK, `posts.id`    | 어떤 게시글에 달린 댓글인지          |
| author_id               | FK, `users.id`    | 댓글 작성자                   |
| parent_comment_id       | FK, `comments.id` | 대댓글 구조 관리, NULL이면 최상위 댓글 |
| content                 | 댓글 내용             | 화면 표시                    |
| created_at / updated_at | 작성/수정 시각          | 정렬, 수정 관리, 히스토리          |

- **사용 시나리오**: 
    - 댓글 작성, 수정, 삭제, 대댓글 구조 지원

## 7️⃣ uploads — 이미지/파일 업로드
| 필드           | 의미                      | 사용 목적            |
| ------------ | ----------------------- | ---------------- |
| id           | PK                      | 업로드 고유 ID        |
| uploader_id  | FK, `users.id`          | 업로더 정보           |
| post_id      | FK, `posts.id`, NULL 가능 | 선택적으로 게시글과 연결    |
| file_path    | 로컬/S3 경로                | 파일 저장 위치         |
| file_name    | 파일 이름                   | 화면 표시, 다운로드 시 사용 |
| content_type | 파일 타입                   | MIME 타입 관리       |
| created_at   | 업로드 시각                  | 히스토리 관리          |

- **사용 시나리오**: 
    - 게시글 이미지 업로드, 관리, 필요 시 게시글과 연결
